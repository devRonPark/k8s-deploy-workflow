AI-Readiness 지도

이 시스템의 설계 원칙은 "결정론 우선, LLM은 나중". 그래서 AI 준비도는 "AI가 얼마나 쓰였나"가 아니라 "AI를 안전하게 꽂을 자리와 안전장치가 얼마나 갖춰졌나"로 읽습니다. 토대는 완성, 안전 하네스는 구축 완료, 그러나 정작 LLM 실행부는 아직 비어 있음 — 이게 현재 좌표입니다.

AI 자리: 갖춰진 것 vs 비어 있는 것

"AI를 꽂을 소켓"은 완성돼 있습니다. 소켓에 들어갈 "전구(LLM 실행부)"만 아직 없습니다.

구축된 AI 안전 하네스

결정론 · 검증됨
  • ✓
    Semantic Task 빌더 — 결정론이 "LLM이 필요한 지점"만 골라 태스크 생성. 한 태스크 = 한 필드.semantic/task_builder.py
  • ✓
    제한된 읽기 도구 4종 — search_code · read_source_range · inspect_entrypoint_script · find_command_target. 한 컴포넌트로 범위 한정.semantic/tools/
  • ✓
    예산 원장 — 호출 수 · 파일 수 · 소스 라인 · 재시도를 누적 강제. 한도 도달 시 이후 호출 거부, 직전 근거는 보존.semantic/budget.py · SemanticToolSession
  • ✓
    결정론적 검증기 — LLM 후보의 근거(grounding)·범위·Secret 유출을 검사해 수용 여부를 최종 결정.semantic/verifier.py
