# Repository Assessment Beta v1.0.0-beta.1

Repository Assessment Beta analyzes a local application repository and writes a review-ready repository assessment. It is not a complete migration agent and does not generate Kubernetes deployment output.

## What v1 Does

- Reads a local repository path.
- Uses deterministic scanner, parser, evidence, and rule outputs.
- Builds `RepositoryUnderstanding` with Confirmed, Unknown, and Conflict findings.
- Writes JSON, Markdown, YAML, and discovery outputs for review.
- Keeps evidence references attached to every Confirmed Fact.

## What v1 Does Not Do

- Kubernetes manifests are not generated in v1.
- No deployment proposal, decision, validation, repair, OpenShell, or Qwen path runs in v1.
- Unknown and Conflict are not treated as engine failures.
- Unsupported repository shapes are reported as limitations instead of guessed.

## Installation

```bash
uv venv --system-site-packages .venv
uv pip install --python .venv/bin/python3 -e .
```

## One-Command Usage

```bash
repository-agent assess tests/fixtures/migration_agent/node-docker
```

Optional output and format:

```bash
repository-agent assess ./sample-app --output .repository-agent/runs/sample --format all
```

## Expected Outputs

The run directory contains:

- `discovery.json`
- `repository-understanding.yaml`
- `repository-assessment.json`
- `repository-assessment.md`

The console summary shows components, execution, structure, build, container, Confirmed, Unknown, Conflict, and Evidence counts.

## Supported Patterns

- Local repository paths.
- `package.json`.
- Dockerfile.
- Docker Compose.
- Maven project metadata and Gradle build-tool hints.
- Spring application configuration.
- .NET project, launch settings, and appsettings metadata.
- Existing Kubernetes manifests and Helm chart metadata as read-only evidence.
- Components, run command, runtime port, container strategy, environment variable names, and dependency hints when supported evidence exists.

## Known limitations

- Multi-service repositories are validation probes, not full supported-scope pass/fail gates.
- Polyrepo layouts are outside v1 scope unless a narrow supported sub-repository path is selected.
- Missing Dockerfile, conflicting ports, and unsupported artifacts remain visible as Unknown, Conflict, or coverage limitations.

## Data safety

Secret values are not written to assessment outputs. The beta preserves names and evidence metadata needed for review, not credential values.
